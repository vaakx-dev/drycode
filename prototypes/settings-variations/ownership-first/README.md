# Ownership-first Settings

A standalone Drycode Settings variation organized by ownership rather than feature area. It keeps Drycode/Core and runtime lifecycle, Starter Chat/Shell composition, and extension-contributed provider configuration visibly separate while preserving the dark starter-chat shell language.

## Run

Serve the repository (for example with `npx serve .`) and open `prototypes/settings-variations/ownership-first/index.html`, or use any static file server from this directory. Lucide UI icons and white provider marks load from their pinned browser CDNs.

## Interactions

Use the left rail to jump between Overview, Extensions, Providers, Runtime, Data & Logs, and About. The graph validation, extension detail rows, provider configuration/test actions, path copy actions, diagnostics actions, log filters, and Runtime Reload confirmation are browser-only in-memory interactions.
