# Domain docs

This repository uses a single-context domain model.

Before exploring, read `CONTEXT.md` and relevant records under `docs/adr/` when they exist. Proceed silently when they do not exist.

Use glossary terms consistently. Surface conflicts with existing architecture decision records rather than silently overriding them.

The domain layout is:

```text
/
├── CONTEXT.md
├── docs/adr/
└── src/
```
